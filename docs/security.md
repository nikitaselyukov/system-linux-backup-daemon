# Безопасность

В проекте предусмотрены базовые меры безопасности:

- использование абсолютных путей для источника и назначения;
- проверка существования исходного каталога;
- автоматическое создание целевого каталога;
- ограничение прав на конфигурационный файл (`640`);
- запрет на повышение привилегий через `NoNewPrivileges=true`.

Что можно усилить в следующей версии:

- добавить отдельного системного пользователя только для резервного копирования;
- внедрить шифрование резервных копий;
- ограничить список допустимых каталогов в конфигурации;
- реализовать удаление старых копий по политике хранения.
