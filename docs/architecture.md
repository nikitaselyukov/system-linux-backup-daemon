# Архитектура SnapshotD

SnapshotD состоит из четырёх основных частей:

1. `snapshot_worker.cpp` — C++-программа, которая выполняет копирование.
2. `snapshotd.service` — systemd-сервис для запуска worker.
3. `snapshotd.timer` — таймер, который вызывает сервис по расписанию.
4. `snapshotctl` — CLI для управления конфигурацией и запуском.

Логика работы:

- конфигурация хранится в `/etc/snapshotd/snapshotd.conf`;
- при срабатывании таймера запускается сервис;
- сервис передаёт worker пути источника и назначения;
- worker создаёт каталог с меткой времени и копирует в него содержимое исходной папки.
